# Корпоративная система реабилитации сотрудников

SSR-платформа для сопровождения сотрудников в рабочем контуре компании: от анкеты и персонального плана до достижений, поощрений, поддержки и управленческой аналитики.

Система рассчитана на постоянную эксплуатацию, а не на демо-режим:
- роли и доступ разграничены по бизнес-процессу;
- все пользовательские сценарии локализованы на русском языке;
- данные и настройки хранятся централизованно в PostgreSQL.

## Что умеет система

### Для сотрудника
- персональный профиль с табельным номером и корпоративной почтой;
- редактирование текущих целей без доступа в админские формы;
- опросник с понятным прогрессом и режимом редактирования;
- план тренировок с датой и временем старта;
- запуск тренировки в разрешенное окно времени;
- персональные напоминания (вкл/выкл, время, дни недели);
- достижения, баллы, рейтинг, поощрения;
- обращения в поддержку и уведомления по ответам/изменениям.

### Для руководителя
- панель отдела с динамикой активности и переносимости;
- карточка сотрудника: достижения, тренировки, полученные поощрения;
- начисление баллов сотруднику с причиной;
- обработка заявок на поощрения;
- уведомления о новых заявках на поощрения.

### Для администратора
- управление пользователями, ролями и доступом;
- управление упражнениями, тренировками, программами, планами;
- настройка достижений и поощрений (включая лимиты получений);
- обработка поддержки и заявок на сброс пароля;
- контроль обратной связи и статусов сопровождения.

## Технологический стек
- Go 1.22+
- `chi` router
- PostgreSQL 14+/16
- HTML Templates (SSR)
- CSS (единая дизайн-система проекта)

## Быстрый старт

1. Поднимите PostgreSQL:
```bash
docker compose up -d
```

2. Подготовьте переменные окружения:
```bash
cp .env.example .env
```

3. При необходимости включите тестовые данные в `.env`:
```env
SEED_DATA=true
```

4. Запустите приложение:
```bash
go run .
```

Система будет доступна на `http://localhost:8080`.

## Тестовые учетные записи (при `SEED_DATA=true`)
- Сотрудник: `10001` / `password`
- Руководитель: `20001` / `password`
- Администратор: `90000` / `password`

## Переменные окружения

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `APP_ADDR` | `:8080` | Адрес HTTP-сервера |
| `DATABASE_URL` | `postgres://rehab:rehab@localhost:5432/rehab_app?sslmode=disable` | Подключение к БД |
| `RUN_MIGRATIONS` | `true` | Автозапуск миграций при старте |
| `SEED_DATA` | `false` | Заполнение демонстрационными данными |
| `ALLOW_SELF_REGISTER` | `false` | Разрешить саморегистрацию сотрудников |
| `COOKIE_NAME` | `rehab_session` | Имя cookie сессии |
| `COOKIE_SECURE` | `false` | Передавать cookie только по HTTPS |
| `SESSION_TTL` | `168h` | Время жизни сессии |

## Миграции и данные
- используется единая актуальная миграция: `migrations/001_init.sql`;
- все изменения схемы и бизнес-логики синхронизированы в backend, шаблонах и seed-данных;
- сиды загружаются из `internal/db/seed.go`.

## Структура проекта

```text
main.go
internal/
  config/          # загрузка конфигурации
  db/              # подключение, миграции, seed
  middleware/      # сессии, логирование, recovery, контекст
  models/          # доменные модели
  site/            # HTTP-обработчики и бизнес-логика
  web/
    templates/     # SSR-шаблоны страниц и partials
    static/        # CSS и статические ассеты
migrations/
  001_init.sql     # единая миграция БД
uploads/           # пользовательские загрузки
```

## Команда разработки

Проверка сборки и пакетов:
```bash
go test ./...
```

Если запускаете в production-контуре, рекомендуется:
- `COOKIE_SECURE=true`;
- отдельная БД и отдельные учетные данные;
- отключенная саморегистрация (`ALLOW_SELF_REGISTER=false`);
- регулярный бэкап PostgreSQL.
