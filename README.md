# Разработка системы спортивной реабилитации сотрудников ГК «Росатом»

**Модуль «Тренировки» для корпоративной платформы сопровождения сотрудников**

Проект реализован как прикладная веб-система на Go (SSR), где в центре не «демо-интерфейс», а полный рабочий цикл: от первичной оценки и назначения программы до контроля прогресса, мотивации и управленческой аналитики.

---

## Выпускная квалификационная работа

Проект выполнен в рамках **выпускной квалификационной работы бакалавриата** и отражает практическую реализацию предложенного подхода.

**Тема ВКР:** Разработка автоматизированной информационной системы подбора и динамической адаптации программ спортивной реабилитации на основе опросников, нормативов и базовых эвристических рекомендаций (модуль «Тренировки» для создания системы спортивной реабилитации сотрудников ГК «Росатом»).

---

## Почему этот проект выделяется

- **Практическая применимость**: архитектура и сценарии рассчитаны на реальную эксплуатацию в корпоративной среде.
- **Персонализация без ручного хаоса**: рекомендации и сопровождение строятся на данных профиля, опросников и динамике выполнения.
- **Ролевая модель управления**: сотрудники, руководители и администраторы работают в едином контуре с разграничением доступа.
- **Прозрачная метрика результата**: прогресс, активность, переносимость нагрузки, достижения и поощрения фиксируются централизованно.

---

## Функциональные возможности

### Роли и ключевые сценарии

| Роль | Что получает пользователь |
|---|---|
| **Сотрудник** | Профиль, цели, опросник, индивидуальный план, запуск тренировок по расписанию, прогресс, достижения, поощрения, поддержка |
| **Руководитель** | Панель подразделения, динамика активности/переносимости, карточки сотрудников, начисление баллов, обработка заявок |
| **Администратор** | Управление пользователями и ролями, контентом тренировок, программами и планами, поощрениями, обратной связью и поддержкой |

### Что реализовано в модуле «Тренировки»

1. Сбор и актуализация входных данных сотрудника.
2. Формирование персонального тренировочного контура.
3. Контроль окна старта и последовательности выполнения.
4. Фиксация выполнения по упражнениям и подходам.
5. Начисление баллов, открытие достижений, влияние на рейтинг.
6. Передача агрегированных метрик в управленческие панели.

## Технологический стек

- **Backend**: Go 1.22+
- **Router**: `chi`
- **DB**: PostgreSQL 14+/16
- **Frontend**: SSR (HTML Templates) + единая CSS дизайн-система
- **Инфраструктура разработки**: Docker Compose

---

## Быстрый старт (локально)

1. Поднимите PostgreSQL:
```bash
docker compose up -d
```

2. Создайте локальную конфигурацию:
```bash
cp .env.example .env
```

3. При необходимости включите сиды:
```env
SEED_DATA=true
```

4. Запустите приложение:
```bash
go run .
```

После запуска сервис доступен на `http://localhost:8080`.

- Администратор: `90000` / `password`

## Конфигурация окружения

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `APP_ADDR` | `:8080` | Адрес HTTP-сервера |
| `DATABASE_URL` | `postgres://rehab:rehab@localhost:5432/rehab_app?sslmode=disable` | Подключение к БД |
| `RUN_MIGRATIONS` | `true` | Автоматический запуск миграций при старте |
| `SEED_DATA` | `false` | Заполнение тестовыми данными |
| `ALLOW_SELF_REGISTER` | `false` | Разрешить саморегистрацию сотрудников |
| `COOKIE_NAME` | `rehab_session` | Имя cookie сессии |
| `COOKIE_SECURE` | `false` | Передавать cookie только по HTTPS |
| `SESSION_TTL` | `168h` | Время жизни сессии |

## Миграции и данные

- Основная миграция: `migrations/001_init.sql`
- Логика заполнения тестовыми данными: `internal/db/seed.go`
- Backend, шаблоны и структура БД синхронизированы под единый функциональный контур

---

## Архитектура проекта

```text
main.go
internal/
  config/          # конфигурация окружения
  db/              # подключение, миграции, seed
  middleware/      # сессии, логирование, recover, context
  models/          # доменные модели

  web/
    templates/     # SSR-шаблоны (pages + partials)
    static/        # CSS и статические ассеты
migrations/
  001_init.sql
uploads/
  .gitkeep
```
