# Система спортивной реабилитации сотрудников ГК «Росатом»

Веб-приложение на Go (SSR) для подбора, сопровождения и динамической адаптации программ спортивной реабилитации сотрудников.  
Фокус текущей реализации: модуль **«Тренировки»** и связанные пользовательские/управленческие сценарии.

## Выпускная квалификационная работа

Проект выполнен как **выпускная квалификационная работа бакалавриата** по теме:

> «Разработка автоматизированной информационной системы подбора и динамической адаптации программ спортивной реабилитации на основе опросников, нормативов и базовых эвристических рекомендаций (модуль „Тренировки“ для создания системы спортивной реабилитации сотрудников ГК „Росатом“)».

## Назначение системы

Система предназначена для промышленного использования в корпоративном контуре и покрывает полный цикл:
- сбор и обновление данных сотрудника;
- формирование и выполнение тренировочного плана;
- учет прогресса, достижений и балльной мотивации;
- обработка заявок, обратной связи и обращений в поддержку;
- управленческий контроль и администрирование контента.

## Функциональные возможности

### Для сотрудника
- профиль с табельным номером и корпоративной почтой;
- редактирование целей и параметров сопровождения;
- опросник с прогрессом и возможностью редактирования ответов;
- индивидуальный план тренировок с датой и временем старта;
- запуск тренировки в разрешенное окно времени;
- достижения, баллы, рейтинг, поощрения;
- напоминания и обращения в поддержку.

### Для руководителя
- панель отдела с динамикой активности и переносимости;
- карточки сотрудников с историей тренировок и достижений;
- начисление баллов сотруднику с указанием причины;
- обработка заявок на поощрения;
- уведомления о новых запросах подразделения.

### Для администратора
- управление пользователями, ролями и доступом;
- управление упражнениями, тренировками, программами и планами;
- настройка достижений и поощрений (включая лимиты);
- обработка поддержки и заявок на сброс пароля;
- контроль качества обратной связи и статусов сопровождения.

## Технологический стек

- Go 1.22+
- `chi` router
- PostgreSQL 14+/16
- HTML Templates (SSR)
- CSS (единая дизайн-система проекта)

## Быстрый старт

1. Запустите PostgreSQL:
```bash
docker compose up -d
```

2. Создайте локальный файл окружения:
```bash
cp .env.example .env
```

3. При необходимости включите тестовые данные:
```env
SEED_DATA=true
```

4. Запустите приложение:
```bash
go run .
```

После запуска приложение доступно по адресу `http://localhost:8080`.

## Тестовые учетные записи (при `SEED_DATA=true`)

- Сотрудник: `10001` / `password`
- Руководитель: `20001` / `password`
- Администратор: `90000` / `password`

## Конфигурация окружения

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `APP_ADDR` | `:8080` | Адрес HTTP-сервера |
| `DATABASE_URL` | `postgres://rehab:rehab@localhost:5432/rehab_app?sslmode=disable` | Подключение к БД |
| `RUN_MIGRATIONS` | `true` | Автозапуск миграций при старте |
| `SEED_DATA` | `false` | Заполнение демонстрационными данными |
| `ALLOW_SELF_REGISTER` | `false` | Разрешить саморегистрацию сотрудников |
| `COOKIE_NAME` | `rehab_session` | Имя cookie сессии |
| `COOKIE_SECURE` | `false` | Передавать cookie только по HTTPS |
| `SESSION_TTL` | `168h` | Время жизни сессии |

## Миграции и данные

- используется единая актуальная миграция: `migrations/001_init.sql`;
- схема, backend-логика, шаблоны и seed-данные синхронизированы;
- демонстрационные данные загружаются из `internal/db/seed.go`.

## Структура проекта

```text
main.go
internal/
  config/          # загрузка конфигурации
  db/              # подключение, миграции, seed
  middleware/      # сессии, логирование, recovery, контекст
  models/          # доменные модели
  site/            # HTTP-обработчики и бизнес-логика
  web/
    templates/     # SSR-шаблоны страниц и partials
    static/        # CSS и статические ассеты
migrations/
  001_init.sql     # единая миграция БД
uploads/           # пользовательские загрузки
```

## Команды разработки

Проверка сборки:
```bash
go test ./...
```

Рекомендации для production-контура:
- `COOKIE_SECURE=true`;
- отдельная БД и отдельные учетные данные;
- отключенная саморегистрация (`ALLOW_SELF_REGISTER=false`);
- регулярное резервное копирование PostgreSQL.
